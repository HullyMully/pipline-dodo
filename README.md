# Детекция событий за столиком в ресторане

Консольное приложение на Python анализирует видео с камеры зала: пользователь задаёт прямоугольную зону столика (ROI), далее **YOLOv8n** ищет людей (класс `person`), и по их положению относительно ROI определяется, **свободен** стол или **занят**. События и состояния записываются в таблицу **Pandas**, на видео рисуется ROI, детекции и таймер; результат сохраняется в `output.mp4`.

## Стек

- **OpenCV** — захват видео, выбор ROI (`cv2.selectROI`), визуализация, запись файла.
- **Ultralytics (YOLOv8n)** — детекция людей, веса `yolov8n.pt` (скачиваются при первом запуске).
- **Pandas** — журнал событий и вывод в консоль.
- **argparse** — аргументы командной строки.

## Установка

Требуется Python 3.10+ (рекомендуется виртуальное окружение).

```bash
cd /path/to/pipline-dodo
python -m venv .venv
source .venv/bin/activate  
pip install -r requirements.txt
```

При первом запуске Ultralytics подтянет `yolov8n.pt` в рабочую директорию (или кэш).

## Запуск

Минимальный вызов:

```bash
python main.py --video /path/to/video.mp4
```

### Аргументы

| Аргумент | Описание |
|----------|----------|
| `--video` | **Обязательный.** Путь к входному видео. |
| `--model` | Путь к весам YOLO (по умолчанию `yolov8n.pt`). |
| `--output` | Выходное видео (по умолчанию `output.mp4`). |
| `--stride` | Обрабатывать детектором каждый N-й кадр (по умолчанию `2`). Увеличьте для ускорения на слабом CPU/GPU. |

Пример с ускорением:

```bash
python main.py --video clip.mp4 --stride 3 --output result.mp4
```

Во время воспроизведения нажмите **q**, чтобы остановить обработку досрочно (часть видео уже будет записана в `--output`).

## Сценарий работы

1. **Первый кадр** — видео останавливается на первом кадре. Открывается окно ROI: выделите мышью прямоугольник столика, подтвердите (как в стандартном `selectROI`: перетаскивание, затем Enter/Space).
2. **Обработка** — с начала ролика для выбранных кадров запускается YOLO, класс только **person** (COCO, id 0).
3. **Занятость ROI** — стол считается **занятым (OCCUPIED)**, если для хотя бы одного человека выполняется хотя бы одно из условий:
   - центр bounding box внутри ROI;
   - **или** не менее 50% площади bbox пересекается с ROI.
4. **Состояния** — **EMPTY** (стол пуст), **OCCUPIED** (стол занят).
5. **События** — в журнал пишется:
   - `INIT` — начальное состояние после первого осмысленного кадра;
   - `STATE_EMPTY` — переход в «стол пуст» (в т.ч. после того как гости ушли);
   - `APPROACH` — **подход к столу**: переход **EMPTY → OCCUPIED**.

## Аналитика

В консоль выводится таблица событий (фрагмент) и метрика:

**Среднее время между моментом, когда стол стал пустым после занятости, и следующим подходом** — то есть для каждой пары переходов **OCCUPIED → EMPTY**, затем **EMPTY → OCCUPIED (APPROACH)** вычисляется длительность интервала в секундах по таймкоду видео; в конце выводится среднее по всем таким интервалам.

Если на видео не было полного цикла «были люди → ушли → пришли снова», среднее не считается, об этом будет сообщение в консоли.

## Визуализация

- ROI: **зелёная** рамка — EMPTY, **красная** — OCCUPIED.
- Текст: текущее состояние, время в текущем состоянии, таймкод видео и номер кадра.
- Детекции людей — оранжевые прямоугольники (на кадрах, где выполнялась детекция; при `stride > 1` на промежуточных кадрах используются последние известные bbox).

## Замечания

- Точность зависит от ракурса, освещения и того, полностью ли ROI покрывает стол.
- Для GPU установите сборку **PyTorch** под вашу систему с [pytorch.org](https://pytorch.org); иначе используется CPU.
- Кодек выходного MP4: `mp4v` (совместимость с OpenCV). При проблемах с проигрыванием перекодируйте через `ffmpeg`.

Анализ результатов:
Среднее время задержки составило 0.44 сек. Столь низкий показатель обусловлен «дребезгом» детекции (flickering): кратковременная потеря объекта моделью YOLOv8n в условиях частичного перекрытия (occlusion) приводит к ложным срабатываниям события «Стол пуст».
Пути улучшения: Внедрение фильтра Калмана для трекинга или использование временного буфера (состояние меняется только после N кадров стабильного обнаружения).
